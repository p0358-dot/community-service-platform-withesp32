# 社区服务平台用户手册

本文档面向社区居民，说明如何启动项目、使用用户端主要功能，并给出常见问题排查思路。

## 1. 使用范围与角色
- **用户端（居民）**：提供场地预约、AI 助手对话、报修提交、通知/便民信息查询、个人中心与我的预约等功能。
- **管理员端（管理人员）**：由系统管理员进行场地/预约审核、用户管理、通知管理、服务管理、数据统计等（见后续“技术方案/设计理念”文档了解整体架构）。
- **设备端（ESP32，可选）**：作为语音入口与物联网控制端（需要协议网关/适配层），把语音意图/控制工具映射到平台业务（预约/报修/设备控制）。

本项目目前为演示/教学用途，后端接口对外**不做身份认证**。

## 2. 启动与访问

### 2.1 安装依赖
在项目根目录执行：
```bash
npm install
```

### 2.2 启动服务
```bash
npm start
```

开发模式（自动重启）：
```bash
npm run dev
```

### 2.3 访问地址
- 用户端：`http://localhost:3000/user/`
- 管理端：`http://localhost:3000/admin/`
- 健康检查：`http://localhost:3000/api/health`
- API 基础前缀：`http://localhost:3000/api/`

## 3. 环境变量与 AI 助手

AI 助手基于阿里云 DashScope API 进行对话调用。你需要在根目录准备 `.env` 文件（或在运行环境中提供同名环境变量）。

参考 `env.example`，常用配置：
- `DASHSCOPE_API_KEY`：DashScope API Key（必需）
- `DASHSCOPE_MODEL`：默认 `qwen-plus`（可选）
- `PORT`：默认 `3000`（可选）
- `NODE_ENV`：默认 `development`（可选）

提示：请勿把真实密钥提交到公开仓库；部署时务必通过环境变量注入。

## 4. 用户端功能使用说明

### 4.1 主页与服务入口
进入用户端后，顶部搜索框与底部工具栏用于快速进入功能。
主页主要入口包括：
- 场地查询
- 场地预约
- 公共设施报修
- 政策通知查询
- 便民服务查询 / 便民点
- 个人中心
- AI 助手（底部“AI助手”）
- 我的预约（底部“我的预约”）

页面采用“页面内切换”的方式呈现内容（不依赖跳转新页面）。

### 4.2 场地查询
操作路径：主页 → 点击“场地查询”
功能说明：
1. 系统拉取场地列表并展示卡片信息（名称、位置、容量、描述、当前状态）。
2. 卡片上会提供预约按钮（当场地为可用状态时可点击）。

后端接口（用于了解实现）：`GET /api/venue/list`

### 4.3 场地预约（表单方式）
操作路径：主页 → 点击“场地预约”
使用步骤：
1. 选择需要预约的场地（下拉列表）
2. 选择预约日期（`input type="date"`）
3. 选择时间段（`time-slot`）
4. 填写备注（可选）
5. 点击“提交预约”

后端接口（用于了解实现）：`POST /api/reservation/create`

常见失败原因：
- 预约时间冲突（同场地同日期同时间段已存在未取消预约）
- 接口路径与前端请求不一致（如果你在本地看到“提交无响应/404”，可检查前端是否指向 `/create` 子路由）

### 4.4 AI 智能助手对话
操作路径：主页/底部 → 点击“AI助手”
使用方法：
- 直接输入文字，例如：
  - “预约乒乓球室 明天下午2点到4点”
  - “查询健身房 今天的预约情况”
  - “取消预约，编号是 3”（或先让助手列出你的预约再取消）
- 语音输入（可选）：
  - 浏览器支持的情况下可点击麦克风图标使用 Web Speech API（语音会转成文本后发送给 AI）
  - 若你接入 ESP32 语音设备，则由网关把设备端识别/控制结果转发到平台 AI 与业务接口

AI 助手能力（当前实现）：
- 识别意图：预约 / 查询 / 取消预约
- 预约流程：自动匹配场地，检查时间冲突，冲突则引导调整时间
- 取消流程：基于你（当前 userId）名下的预约列表进行取消

后端关键接口：
- 对话：`POST /api/ai/chat`
- 取消预约（AI 间接触发）：`POST /api/ai/cancel`
- 获取状态/测试：`GET /api/ai/status`、`GET /api/ai/test`

### 4.5 我的预约（预约记录查看与取消）
操作路径：底部 → 点击“我的预约”
功能说明：
1. 页面会加载你的预约列表，并按状态分组展示（待处理/历史等）。
2. 对于仍可取消的预约，会提供“取消预约”按钮。
3. 点击取消后刷新列表。

后端接口（用于了解实现）：
- 获取我的预约：`GET /api/reservation/user/:userId`
- 取消预约：`DELETE /api/reservation/:id`

### 4.6 设备报修（提交与查看记录）
操作路径：主页 → “公共设施报修”
功能说明：
1. “报修记录”会展示你提交的工单列表。
2. “提交报修”需要填写：
   - 故障标题（必填）
   - 故障详情（必填）
   - 选择故障场地（必填）
   - 优先级（可选）
3. 提交成功后重新加载记录。

后端接口（用于了解实现）：
- 获取我的报修：`GET /api/repair/user/:userId`
- 创建报修：`POST /api/repair/create`

### 4.7 政策通知查询与便民服务
操作路径：
- 主页 → “政策通知查询”
- 主页 → “便民服务查询”

功能说明：
- 通知列表：从已发布通知加载并支持筛选
- 便民服务：从服务列表加载并展示评分/状态

后端接口（用于了解实现）：
- 通知（已发布）：`GET /api/notice/published`
- 通知（管理员）：`GET /api/notice/list`
- 服务列表：`GET /api/service/list`

## 5. 常见问题排查

### 5.1 AI 服务不可用
可能原因：
- `.env` 中 `DASHSCOPE_API_KEY` 未配置或不正确
- 外网访问受限（服务器无法访问 DashScope）
- 模型/接口参数不匹配

处理建议：
- 检查 `.env` 是否存在并重启服务
- 用浏览器打开 `GET /api/ai/test` 看接口是否连通

### 5.2 预约/提交报修失败
可能原因：
- 请求体缺少必填字段
- 时间冲突或参数格式不正确
- 前端调用的接口路径与后端路由不一致

处理建议：
- 打开浏览器开发者工具查看请求路径与响应码
- 与后端路由对照（例如预约创建是否需要 `/create`）

### 5.3 跨域/加载失败
本项目 CORS 配置较宽松用于开发测试。若部署到特定域名环境，可能仍需调整 CORS 与静态资源路径。

## 6. （可选）ESP32 语音设备接入

如果你希望通过 ESP32 语音设备来完成社区事务（例如预约/查询/取消预约、提交报修、触发设备控制），通常需要一个“协议网关/适配层”：

1. ESP32 设备端通过 WebSocket 或 MQTT + UDP 与网关建立会话，并使用 MCP 工具调用表达意图/控制请求。
2. 网关侧把 MCP `tools/call` 映射到本仓库后端已有的 HTTP API，再把执行结果回填给设备侧。

本仓库当前的 `backend/` 主要提供 HTTP API 与 DashScope AI 对话接口；并不直接承载 ESP32 的 WebSocket/MQTT/MCP 会话与流式语音处理。
因此建议按以下资料先行：

- 阅读：`docs/ESP32硬件接入指南.md`
- 按你的通信方式选择：
  - WebSocket：按 `esp32/docs/websocket.md` 验证握手与消息类型
  - MQTT + UDP：按 `esp32/docs/mqtt-udp.md` 验证 MQTT hello 与 UDP 音频通道

## 7. 常见问题（设备端）

- 设备连不上网关：检查设备所在网络是否能访问网关地址（WebSocket 可达 / MQTT broker 可达 / UDP 端口可达），以及 `Authorization` 令牌是否配置正确。
- 工具调用失败：通常是 `tools/list` 返回的工具名与你网关映射表不一致，或 HTTP API 参数缺失导致后端返回 400/500。
